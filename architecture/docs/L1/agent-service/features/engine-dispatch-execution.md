# Engine Dispatch & Execution — Feature Inventory (AS-L1-F33..F39)

> Module: Engine Dispatch & Execution (Layer 5a per ADR-0140).
> Sovereign for: EngineRegistry strict matching (Rule R-M.a/.b), ExecutorAdapter lifecycle, third-party Agent adapter, sub-agent executor boundary, EngineHookSurface emission (HookPoint events go to Layer 4 — Layer 5a does NOT invoke RuntimeMiddleware directly), stream/partial-result handoff, adapter capability + version governance.
> Does NOT own: model/tool translation (lives in Layer 5b — Translation & Tool-Intercept), runtime governance (lives in Layer 4 via HookPoint), Run aggregate state.

> **Status (ADR-0159 pure rebuild):** this Layer-5a inventory is the **pre-ADR-0159 agent-service engine-dispatch DESIGN** (`status: proposed`). The engine was consolidated into `agent-runtime`: the current shipped dispatch is `EngineDispatcher` → `AgentRuntimeHandler` (routed by `agentId`; unknown `agentId` → terminal `AGENT_ID_INVALID`; single `control` write authority), verified by `agent-runtime`'s `EngineDispatcherTest` / `EngineClosedLoopIntegrationTest`. The `EngineRegistry` / `ExecutorAdapter` / `EngineEnvelope` / `EngineHookSurface` / `HookPoint` / `RuntimeMiddleware` design named throughout this inventory is **RETIRED / design_only** — no such Java type exists (`engine-envelope.v1.yaml` / `engine-hooks.v1.yaml` are `design_only`).

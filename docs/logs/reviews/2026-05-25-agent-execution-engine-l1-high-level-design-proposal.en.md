# Architecture Review Proposal: agent-execution-engine L1 Design Draft

> **Date:** 2026-05-25
> **Target:** L1 design draft. This document is not a final L0 release authority.
> **Authority anchors:** `agent-execution-engine/ARCHITECTURE.md`, `agent-service/ARCHITECTURE.md`, `docs/contracts/contract-catalog.md`, ADR-0112.

## 1. Scope Calibration

This proposal separates the current engine contract from accepted-forward and exploratory work. The engine direction remains valid: stateless execution, server-owned read-modify-write closure, explicit yields, and a future declarative developer experience. The correction is maturity labeling.

| Track | Status | Contract surface | May claim now | Must not claim now |
|---|---|---|---|---|
| Current L0/L1 shipped surface | Current | `ExecutorAdapter`, `GraphExecutor`, `AgentLoopExecutor`, `EngineHookSurface`, `EngineRegistry`, `EngineEnvelope`, `SuspendSignal`, `com.huawei.ascend.service.engine.spi.StatelessEngine` | The platform has a synchronous stateless engine SPI and existing engine registry/envelope surface | Runtime-owned dynamic compilation, generated bytecode packaging, production package bootloading, or debugger services |
| Accepted-forward yield migration | Forward design | ADR-0112 | `Mono<StateDelta>` and value-carried yield are accepted migration direction | Active Java signatures have not moved until catalog, DFX, module metadata, Java code, and tests move together |
| Declarative / generated engine tooling | W2+ exploratory | No current runtime contract | Candidate future design space | Immediate W0/W1 release scope, production security guarantees, or package format compatibility |

## 2. Current Contract Truth

`agent-execution-engine` currently owns the engine-side contract and dispatch surfaces under `com.huawei.ascend.engine.*`:

```text
agent-execution-engine/src/main/java/com/huawei/ascend/engine/
├── spi/
│   ├── ExecutorAdapter.java
│   ├── GraphExecutor.java
│   ├── AgentLoopExecutor.java
│   ├── EngineHookSurface.java
│   └── EngineMatchingException.java
├── orchestration/spi/
│   ├── Orchestrator.java
│   ├── Checkpointer.java
│   ├── RunContext.java
│   ├── TraceContext.java
│   ├── SuspendSignal.java
│   ├── RunMode.java
│   └── ExecutorDefinition.java
└── runtime/
    ├── EngineRegistry.java
    └── EngineEnvelope.java
```

`agent-service` owns the current pure-function service-to-engine SPI:

```text
agent-service/src/main/java/com/huawei/ascend/service/engine/
├── adapter/InMemoryStatelessEngine.java
└── spi/
    ├── AgentInvokeRequest.java
    ├── StateDelta.java
    └── StatelessEngine.java
```

The current `StatelessEngine` signature is synchronous and returns `StateDelta`. ADR-0112 describes a future value-based yield migration; it does not replace the current Java interface in this release transaction.

## 3. Scenario View

| Scenario | Current handling | Forward handling |
|---|---|---|
| Embedded service-to-engine execution | `agent-service` projects request state, calls `StatelessEngine`, receives immutable `StateDelta`, then owns persistence and state transitions | Reactive return type may be introduced with ADR-0112-aligned contract changes |
| Graph and ReAct dispatch | `EngineRegistry` resolves `EngineEnvelope` to a registered `ExecutorAdapter`; mismatches fail through `EngineMatchingException` | Additional executor families can register through the same envelope discipline |
| Yield and suspension | `SuspendSignal` remains the checked-exception suspension primitive; `StateDelta.RunStatusTransition.YIELDED` is a typed hint | ADR-0112 may carry yield as value once code, schema, and tests move together |
| Declarative configuration | Design candidate only | Requires schema, package, DFX, security, and gate contracts before implementation |
| Developer debugging and mocking | Out of current release scope | W2+ design candidate after process and security contracts exist |

## 4. Logical View

The logical split is:

| Boundary | Owner | Responsibility |
|---|---|---|
| Read-modify-write closure | `agent-service` | Load Run/Task/Session state, project context, enforce tenant ownership, persist results |
| Stateless compute | `agent-service.engine.spi.StatelessEngine` plus engine executors | Compute `StateDelta` from explicit inputs without owning persistence |
| Engine selection | `agent-execution-engine.runtime.EngineRegistry` | Resolve `EngineEnvelope` to the only valid adapter |
| Cross-cutting hooks | `agent-middleware.spi.RuntimeMiddleware` + `HookPoint` | Observe policy, telemetry, memory, quota, and failure boundaries through hooks |
| Client callback | `agent-bus.spi.s2c` | Carry server-to-client callback envelopes and responses |

No proposal here introduces `com.huawei.ascend.agent.engine.*`. New engine-owned packages must stay under `com.huawei.ascend.engine.*`, or a future ADR must explicitly supersede the package-root authority.

## 5. Capability Boundary Matrix

| Capability | Current home | Current status | Engine may do | Engine must not do | Forward work |
|---|---|---|---|---|---|
| Memory | `GraphMemoryRepository` plus runtime hooks | SPI only; sidecar adapter future | Consume injected memory views and emit memory write intents | Read or write the memory store directly | W2 filters plus Graphiti adapter |
| Skills | `SkillCapacityRegistry`, `ResilienceContract`, engine tools | Partial W1 governance | Execute pre-approved tool abstractions | Resolve tenant capacity internally | W2 scheduler admission |
| Sandbox | policy YAML plus future executor boundary | schema shipped; enforcement deferred | Request sandbox execution through service-owned boundaries | Compile or run generated code outside policy | W2 `SandboxExecutor` design |
| Knowledge | none | W3+ deferred | Consume injected snippets only | Own a knowledge service or ontology | W3+ ADR |

## 6. Dynamic Compilation and Package Work

Dynamic Java compilation, generated component source, debugger/mock registries, classloader bootloading, and `.apg` archives are W2+ exploratory items. They are not current engine behavior and are not prerequisites for the L0 release.

Before any implementation, the release transaction must add a dedicated contract family:

| Contract | Purpose |
|---|---|
| `docs/contracts/component-registry.v1.yaml` | Registry shape, lock semantics, and component metadata |
| `docs/contracts/agent-package.v1.yaml` | Package manifest, allowed entries, checksum/signature fields |
| `docs/contracts/generated-component-safety.v1.yaml` | Static analysis, dependency, and deterministic replay constraints |
| `docs/governance/apg-signing-policy.yaml` | Signing authority, key custody, rotation, revocation |
| `docs/governance/classloader-lifecycle.yaml` | ClassLoader lifetime, unloading, resource caps, rollback |

`lockRegistry()` may reduce runtime mutation surface in a future design. It does not prove loaded code is authorized, reproducible, tenant-isolated, resource-bounded, or safe.

## 7. Open-Source Baseline Adoption Guard

`openJiuwen/agent-core-java` is a candidate physical baseline, not a release dependency. No production implementation may depend on it until an adoption ADR records:

| Required ADR field | Minimum content |
|---|---|
| repository baseline | repository URL, version or commit, and source snapshot policy |
| license and provenance | license compatibility, notices, and provenance checks |
| package strategy | direct import, fork, wrapper, or rewrite decision |
| ownership cadence | update cadence, patch ownership, and divergence policy |
| forbidden dependencies | dependencies disallowed by platform governance |
| gap map | mapping from `WorkflowEngine` / `ReActEngine` to stateless execution and yield contracts |

## 8. Process View

Current execution:

```text
agent-service
  -> builds AgentInvokeRequest
  -> calls StatelessEngine.execute(request)
  -> receives StateDelta
  -> applies typed run transition hint
  -> persists Run/Task/Session/Memory intents through service-owned stores
```

Forward ADR-0112 migration:

```text
agent-service
  -> calls future reactive engine signature
  -> receives value-carried StateDelta / yield signal
  -> persists only through service-owned read-modify-write closure
```

Exploratory generated-component pipeline:

```text
design candidate only
  -> requires adoption ADR
  -> requires safety contracts
  -> requires package and signing policy
  -> requires gates and tests
  -> may then enter W2+ implementation planning
```

## 9. Acceptance Criteria

This L1 proposal is acceptable only if:

1. Current package roots match the Java corpus and contract catalog.
2. `StatelessEngine` remains the current service-owned SPI until a future ADR changes it.
3. ADR-0112 value-based yield is described as accepted-forward, not shipped.
4. Dynamic compiler, generated bytecode, `.apg`, debugger, and mock registry work is W2+ exploratory.
5. Capability boundaries prevent the engine from owning memory, skill capacity, sandbox policy, or knowledge services.
6. Any future implementation wave lands schema, DFX, catalog, Java, tests, and security gates in the same transaction.

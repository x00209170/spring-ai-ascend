## What was restored / created

### C29 — ResilienceContract

| File | Role |
|------|------|
| `agent-runtime/.../resilience/ResilienceContract.java` | SPI interface: `resolve(operationId) → ResiliencePolicy` |
| `agent-runtime/.../resilience/ResiliencePolicy.java` | Immutable record `(cbName, retryName, tlName)` |
| `agent-runtime/.../resilience/YamlResilienceContract.java` | Map-backed impl; dev → DEFAULT_POLICY on unknown; research/prod → throws |
| `agent-runtime/.../resilience/ResilienceContractTest.java` | Layer 1: 6 assertions |
| `agent-runtime/.../resilience/ResilienceContractIT.java` | Layer 2: resolved cbName maps to real Resilience4j CircuitBreakerRegistry |

Why not deleted again: Resilience4j has no `operationId → policy-triple` resolver. When W2
lands the LLM gateway, that routing layer is needed. The C2 deletion treated this as a
"throwing sentinel" but the routing semantics had no OSS equivalent.

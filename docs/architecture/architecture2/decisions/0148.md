# 0148. Wave 0 spike results — Structurizr workspace authority feasibility

Date: 2026-05-27

## Status

accepted

## Context

ADR-0147 §spike-gate requires Wave 0 to validate four technical
assumptions on a single module (agent-bus, smallest L1 by
ARCHITECTURE.md line count = 182) before scaling to Wave 1+.
This ADR records the measured evidence.

The spike scope is intentionally minimal:
- architecture-spike/tools/architecture-workspace-spike/ (Maven
  module outside the reactor)
- One DSL fixture covering agent-bus module + 1 capability + 1
  function point + 1 SPI + 3 relationships (contains, implements,
  declares_spi)
- One negative fixture (capability missing required saa.* properties)
- Three JUnit 5 tests: positive parse+validate, negative violation
  detection, 3× idempotent emission via SHA-256 comparison

## Decision

Wave 0 spike decision: PROCEED to Wave 1.

All 4 gates measure PASS. ADR-0147 authority direction is
validated. The 7-wave plan (historical session plan, local-only)
proceeds.

Spike artifacts under `architecture-spike/` remain in the tree
as a regression suite for the design decisions until Wave 7
cleanup explicitly removes them.

## Consequences

Positive:
- ADR-0147 authority direction is empirically validated; the
  7-wave plan proceeds.
- Three surprise findings (CustomElement subclass relationship;
  hierarchical scope behavior; structurizr-export transitivity)
  caught at Wave 0 before they propagated into 600+ generated
  DSL elements in Wave 3.
- The spike artifacts under `architecture-spike/` serve as
  regression fixtures for the production ProfileValidator that
  lands in Wave 1.

Negative / residual:
- One-time first-build cost of 3m30s to download Structurizr +
  Jackson + SnakeYAML + JUnit. CI machines will pay this once
  per fresh container.
- The 4-gate spike pattern adds a 1-2 week wall-clock cost vs.
  "just start Wave 1". User's standing rule
  "never estimate, always measure" justifies this cost.

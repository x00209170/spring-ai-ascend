# 0068. Layered 4+1 and Architecture Graph as Twin Sources of Truth

Date: 2026-05-14

## Status

accepted

## Context

At W1 entry the repo carries 67 ADRs, 32 active engineering rules,
47 gate-script rules, and 54 enforcer rows. Structured side-files exist
(enforcers.yaml, architecture-status.yaml, competitive-baselines.yaml,
per-module module-metadata.yaml, docs/dfx/*.yaml, principle-coverage.md,
posture-coverage.md, rule-history.md, whitepaper-alignment-matrix.md)
but there is no JOIN between them. ADR-to-ADR relationships (supersedes,
extends, relates-to) live only in prose body text. No machine-readable
graph exists that an LLM could load to traverse
"principle → rule → enforcer → test → artefact"
without reading every ADR.

Continuing to add substring gate rules has reached diminishing returns:
we already maintain 24 self-tests whose only purpose is to police the
gate rules themselves. The slope is asymptotic. The empirical case for
a structural answer is stronger than the doctrinal one — the doctrine
predicts the failure mode the defect taxonomy already exhibits.

## Decision

Adopt two coupled L0 sources of truth:

1. Layered 4+1 corpus (human-facing).
   Every architectural artefact declares `level` and `view` front-matter.
   Root ARCHITECTURE.md is L0; agent-*/ARCHITECTURE.md is L1; deep
   technical designs in docs/L2/ are L2. Each level organises content
   under the 4+1 view headings (L2 may omit irrelevant views). All
   proposals in docs/logs/reviews/ declare `affects_level` + `affects_view`.
   Phase-released L0/L1 artefacts are frozen — further edits flow
   through docs/logs/reviews/.

2. Architecture knowledge graph (machine-facing).
   docs/governance/architecture-graph.yaml is the single machine-readable
   join of all architectural relationships. The file is GENERATED by
   gate/build_architecture_graph.sh from authoritative inputs
   (enforcers.yaml, architecture-status.yaml, principle-coverage.yaml,
   module-metadata.yaml, docs/adr/*.yaml). It is NOT hand-edited. Edge
   classes: principle → rule, rule → enforcer, enforcer → test,
   enforcer → artefact, capability → test, module → module
   (allowed/forbidden), adr → adr (supersedes / extends / relates_to),
   (level, view) → artefact. The supersedes/extends sub-graph is a DAG.
   Every endpoint resolves to a real graph node or file path. The
   build script is idempotent.

3. ADR corpus migrates from .md to .yaml.
   Each ADR becomes a YAML file with structured metadata + relations as
   keys, and free-form prose (context, decision, consequences, rationale)
   as YAML block scalars (`|`). The shape is fully machine-parseable
   while preserving paragraph-level rationale — no institutional memory
   lost. Migration is mechanical (script + manual rationale capture for
   ~20–25 ADRs whose prose carries non-derivable context). ADR-0068
   (this file) is the first native YAML ADR and the canonical example.

4. Code-as-Contract preserved (Rule R-C.a).
   This ADR's enforcers (Rule G-1 sub-clause .a, Rule G-1 sub-clause .b, Gate Rules 37–40,
   ArchitectureLayeringTest, gate/build_architecture_graph.sh) ship
   in the same PR as this ADR. The migration of the 67 prior .md ADRs
   to .yaml is also part of the same PR — at landing time there is
   exactly one ADR file shape in the repo.

## Consequences

- CLAUDE.md gains Rule G-1 sub-clause .a (Layered 4+1 Discipline) and Rule G-1 sub-clause .b
  (Architecture-Graph Truth). Engineering-rule lifecycle after this wave:
  18 active (Rules 1–6, 9–10, 20–21, 25, 28–34), 15 deferred (Rules 7, 8,
  11, 13–19, 22–24, 26, 27), 1 retired (Rule 12) — 34 total introduced.
  Layer-0 governing principles: 4 (P-A, P-B, P-C, P-D).

- "Post-impl note (2026-05-17 six-module materialization PR):
  docs/governance/architecture-graph.yaml was regenerated by
  bash gate/build_architecture_graph.sh after the 5 new modules
  (agent-client, agent-bus, agent-middleware, agent-execution-engine,
  agent-evolve) landed. New count: 269 nodes / 377 edges. Rule R-L
  (architecture_graph_idempotent) reconfirmed; the build script remains
  byte-deterministic over the same inputs. This ADR's commitment to a
  machine-readable graph index survived the first significant module
  decomposition without manual edits."

- gate/check_architecture_sync.sh gains Gate Rules 37–40
  (front-matter completeness, graph well-formedness, review-template
  front-matter, enforcer reachability from a principle).

- Every existing docs/adr/NNNN-*.md is replaced by docs/adr/NNNN-*.yaml.
  External links to .md ADRs (release notes, ADR README, prose citations)
  are updated mechanically. docs/adr/README.md regenerates from the YAML
  corpus.

- Root ARCHITECTURE.md is reorganised under the 4+1 view headings at L0
  scope. agent-service/ARCHITECTURE.md and agent-service/ARCHITECTURE.md
  are reorganised at L1 scope. A docs/L2/ directory is introduced
  (initially with placeholder README and at least one seed L2 doc).

- docs/logs/reviews/_TEMPLATE.md is introduced. Every new review proposal
  declares affects_level / affects_view.

- docs/governance/principle-coverage.md is replaced by .yaml form (with a
  generated .md view) so the graph generator can consume it directly.

- The 4-shape defect model (REF-DRIFT, HISTORY-PARADOX, PERIPHERAL-DRIFT,
  GATE-PROMISE-GAP) gains a structural answer for the dominant family:
  text-drift defects are caught by graph-edge integrity, not by regex.

- Onboarding for new contributors and new AI sessions becomes "read the
  graph summary" first, then drill into prose at the chosen (level, view)
  — replacing the current "scan 67 ADRs and hope to notice the relevant
  ones."

- Cost: ~7 working days for the initial wave (mechanical migration +
  generator + gates + drift-defect regression demo). Recurring cost is
  near zero — the graph regenerates from inputs that already exist.
